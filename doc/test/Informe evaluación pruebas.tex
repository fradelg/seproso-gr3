\chapter{Introducción.} 

\section{Propósito.}

\indent \indent Este documento presenta el Informe de Evaluación de pruebas para resumir los resultados y realizar las evaluaciones pertinentes basadas en esos resultados.


\section{Ámbito.}

\indent \indent Este informe describe los resultados  de las pruebas de SEPROSO, en términos de la cobertura de pruebas y análisis de defectos. Los datos utilizados son los que pueblan la base de datos asociada a la aplicación, y el documento relacionado de resultados. Este documento tiene cómo objetivos:\\
\begin{itemize}
	\item Evaluar la aceptabilidad del rendimiento de la aplicación SEPROSO.
	\item Evaluar la aceptabilidad de las pruebas.
	\item Identificar mejoras para incrementar la calidad de las pruebas.
\end{itemize}

Las pruebas demuestran que los errores más comunes que se han producido son:
\begin{itemize}
	\item Problemas de coherencia en la secuenia de búsqueda en la bse de datos. Simplemente solucionables con cambiar la sentencia de búsqueda.
	\item Problemas de visualización de los diferentes navegadores, corregido usando un estilo CSS común en la web.
	\item Problemas ante usuarios mal itencionados que utilizan entradas erróneas al sistema. Corregido con control de errores.
\end{itemize}

\section{Tipos de errores}
\subsection{Integridad de los datos}

\indent \indent Error en casos de prueba de salidad de datos:
\begin{itemize}
	\item Impacto potencial: Alto, necesario para la aplicación un correcto funcionamiento de la salida de datos.
	\item Impacto relativo: Superior a otros errores potenciales.
\end{itemize}

\subsection{Efecto de borde}

\indent \indent Este error se produce por la salida de rango de los parámetros de las funciones. En la aplicación se controlan los rangos de los parámetros pero aú con esto puede producir errores si están mal limitados.

\chapter{Cobertura de pruebas basadas en el código ejecutado}

\indent \indent Las pruebas se han realizado sobre un prototipo con el 84 \% de código preparado, para evaluar la bondad de ejecución de la aplicación y verificar la calidad del código desarrollado. Posteriormetne en las siguientes iteraciones se realizarán las pruebas sobre la totalidad del código desarrollado. La finalidad de las pruebas es obtener resultados sobre un prototipo con más del 90 \% de desarrollo completado.
