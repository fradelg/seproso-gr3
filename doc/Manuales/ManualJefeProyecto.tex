\chapter{Introducción.} 

\section{Propósito.}

\indent \indent En este documento se presenta la funcionalidad básica de SEPROSO desde el punto de vista de un jefe de proyecto que utiliza la herramienta. El objetivo de este manual es proporcionar al jefe de proyecto la información necesaria para utilizar la herramienta SEPROSO.


\section{Ámbito.}

\indent \indent Mediante esta práctica herramienta Web se pretender dar capacidad para

\begin{itemize}
	\item Planificar sus proyectos software.
  \item Seguir y monitorizar el desarrollo del proyecto.
  \item Controlar y aprobar los informes de trabajo de los desarrolladores.
  \item Conocer en cada momento la situación de un trabajador.
\end{itemize}

\section{Definiciones.}

\indent  \indent Véase el Glosario.

\subsection{Definir nuevo modelo de fases.}

\indent \indent La definición del modelo de proceso de un proyecto se realiza desde la pestaña de \textit{PROYECTO}, y en su submenu de \textbf{Fases}. para la determinación del modelo se debe seleccionar una de las plantillas predeterminada, o bien determinar el nuevo modelo predefinido de forma manual. Podemos ver el menú en la siguiente imagen \ref{modelo}.\\

\figura{Images/modelo.png}{height=0.8\linewidth, angle=90}{Menú para determinar nuevo Modelo de proceso .}{modelo}

Si se quiere definir un nuevo modelo de fases se precisa seleccionar la opción de \textbf{Definir Personalizada}. Tras ello obtendremos las opciones de ir añadiendo nuevas fases al modelo de proceso actual para el Proyecto seleccionado. En la imagen \ref{gestion} vemos cómo es el menú de determinación de las diferentes fases inicialmente. Se irán añadiendo las diferentes fases usando el botón de \textbf{Nueva Fase}. Finalmente podremos obtener un modelo similar al siguiente ejemplo \ref{modelo_ejem}, en el que se han añadido solo 3 fases: Ingenieria de Software, Desarrollo y pruebas y la fase de Implantación . \\

\figura{Images/gestion_fases.png}{width=1.0\linewidth}{Definir modelo de fases de forma manual. Añadir fases del proyecto.}{gestion}

\figura{Images/fases_completo.png}{width=1.0\linewidth}{Ejemplo de modelo de proceso generado manualmente.}{modelo_ejem}

Se puede implantar el nuevo modelo de proceso creado cómo una nueva plantilla por defecto generada para los siguientes proyectos que se creen. Para ello hay que pulsar sobre la opción de \textbf{Establecer como plantilla}. Obtendremos la siguiente plantilla a rellenar \ref{nueva_plantilla}. Dotando de nombre a la plantilla y una breve descripción podremos almacenar el resultado.

\figura{Images/nueva_plantilla.png}{width=1.0\linewidth}{Generar nueva plantilla de modelo.}{nueva_plantilla}

Tras rellenar la plantilla pulsaremos la opción \textbf{Guardar}, y obtendremos un nuevo modelo de plantilla que se puede utilizar en los siguientes proyectos que se implementen. Podemos ver el resultado aquí \ref{guardar_plantilla}.

\figura{Images/guardar_plantilla.png}{width=1.0\linewidth}{Ejemplo de nueva plantilla de modelo de proceso para un proyecto.}{guardar_plantilla}

\subsection{Iniciar Proyecto.}

\indent \indent Una vez definido el modelo de proceso de un proyecto se deben definir las actividades y los trabajadores que lo componen. Para ello visitar las opciones del manual: \ref{acts} y \ref{trabs}. Una vez comenzado un proyecto no se podrá volver a editar la división en fases ni las actividades que lo componen.\\

\newpage
\subsection{Crear Actividad.}

\indent \indent La definición de actividades de un proyecto viene precedida por la creación del modelo de proceso (ver \ref{Pry}) del proyecto. Una vez establecido el modelo de fases del proyecto, se pueden definiar las actividades que compondrá cada una de las fases del proyecto. Para ello accedemos al menú \textit{PROYECTO}, y a la opción marcada cómo \textbf{Actividades}.\\

La creación de una nueva actividad se realiza con el submenú inferior de la herramienta, ver \ref{menu_crear_act} para observar la opción de creación de actividades. Una vez iniciada la creación de una actividad, veremos la primera plantilla a rellenar \ref{Actividades1}, dónde se indica la fase a la cuál se asocia la Actividad, las posibles actividades precedentes y el Trabajador asociado a la actividad. Todos los campos son obligatorios, aunque la actividad puede no tener actividaes predecesoras, dejando entonces ese campo sin marcar.\\

Una vez determinada la posición temporal de la actividad, accedemos a la siguiente fase de creación pulsando \textbf{Siguiente}. En esta nueva plantilla de configuración, se indicará el tipo de actividad de una lista desplegable, el artefacto asociado a la finalización de la actividad y la fecha de inicio de la propia actividad. Así mismo se incluyen el número de horas/hombre que conllevará su realización y el Nombre que tendra la actividad. Además se permite incluir una breve descripción, ver imagen anexa: \ref{Actividades2}.


\figura{Images/activities1.png}{width=0.6\linewidth}{Crear nueva actividad.}{menu_crear_act}
\figura{Images/activities.png}{width=1.1\linewidth}{Crear nueva actividad: plantilla situación.}{Actividades1}
\figura{Images/activities2.png}{width=1.1\linewidth}{Crear nueva actividad: plantilla actividad.}{Actividades2}

\newpage
\subsection{Modificar Actividad.}

\indent \indent Desde el menú de gestión de actividades se permite editar una actividad ya determinada. Se realiza mediante el botón anexo a cada actividad \textbf{Editar}. Con ello permitimos modificar la plantilla de la actividad creada precedentemente.\\

\subsection{Finalizar Actividad.}

\indent \indent El jefe de proyecto es el único que puede dar por finalizada una actividad, ésto se realiza con el botón Finalizar anexo a las actividades y que indicará que la actividad que estaba en curso ha finalizado. Es necesaria una confirmación de la acción antes de proseguir.\\

\chapter{Informe de Actividades.}

\indent  \indent Mediante esta opción del menú se controla la gestión de los informes de actividades realizados por los trabajadores. Mediante una plantilla se selecciona el trabajador, la actividad asociada a él y el informe que ha generado para poderlo observar. A continuación se muestra un ejemplo concreto de su uso \ref{u_act} y de la plantilla \ref{p_act}.\\

Una ver obtenido un informe de actividades se puede \textbf{rechazar}, o \textbf{supervisar} por parte del Jefe de proyecto, mediante un submenu en la parte inferior del informe. Normalmente la mayoría de informes se supervisarán, y el sistema mostrará un mensaje de confirmación de la acción. Si un informe ha sido supervisado por el jefe de proyecto, la siguiente o sucesivas veces q se consulte puede ser \textbf{aprobado}, y por ello dicho informe se almacenará para su visualización cómo actividades aprobadas. Igualmente el informe de un trabjador puede ser \textbf{rechazado} con sus respectiva opción.\\

\figura{Images/gestion_act.png}{height=0.8\linewidth, angle=90}{Plantilla de gestión de actividades.}{p_act}
\figura{Images/ejemplo_act.png}{width=1.1\linewidth}{Ejemplo de informe de gestión de una actividad concreta.}{u_act}
